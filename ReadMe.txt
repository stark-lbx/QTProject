Client:客户端功能
	用户：注册、登录、查找
	好友：添加（响应添加）、删除、聊天
	文件：上传、下载、分享；（刷新、新建目录、删除目录、删除文件、重命名、移动）
Server:服务端功能
	处理客户端各类请求

持续优化方向：
	1.池式结构管理：
		线程池（管理多线程上传文件时，线程的频繁建销）
		内存池（管理mkPDU频繁的建立开销）
		连接池（管理数据库的连接）
	2.安全性设计：
		登录功能优化（验证码登录）
		密码加密（客户端与服务器之间的加密、服务器与数据库之间加密）
	3.文件方向优化：
		不同类型文件的查看
		虚拟文件系统（可以借鉴采用余庆的fastDFS分布式文件系统架构）
		上传文件
			分片上传
			断点续传
			秒传功能（记录每个文件的哈希值md5）
		分享文件链接
